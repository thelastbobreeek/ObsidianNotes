Класс в [[CSharp|C#]] - это **шаблон** для создания **объектов**. Чертеж, описывающий **данные**(поля) и **поведение**(методы). Сам по себе класс это **не** **объект**, но по нему можно создавать сколько угодно **объектов**.
- - -
## Класс - ссылочный тип
Класс - это [[Ссылочные и значимые типы|ссылочный тип]]. Это значит, что при **копировании** объектов из одного **поля** в **другое**:
```cs
Player a = new Player();
Player b = a;
```
Оба поля указывают на **один** **и тот же объект** в [[Стэк и куча|куче]]. 
Это значит:
- при передаче поля в качестве **аргумента** в метод, передаётся **ссылка**, а не объект
- при изменении состояния **внутри** **метода** изменяется **оригинал**
- [[Сборщик мусора]] управляет временем жизни

В [[Стэк и куча|стэке]] хранится **ссылка**, а в [[Стэк и куча|куче]] - сам **объект**.
## Класс как инкапсуляция
Класс - это прежде всего **контроль** **над** **инвариантами**.
Так, например, класс должен **гарантировать**, что он **не** **сломается** и **не** **вызовет** ошибку. 
Например, плохой код:
```cs
public int Health;
```
Лучше же написать так:
```cs
private int _health;

public int Health => _health;

public void TakeDamage(int damage)
{
    if (damage <= 0)
        return;

    _health -= damage;

    if (_health < 0)
        _health = 0;
}
```
Таким образом, грамотно написанный класс будет **гарантировать**, что его состояние:
- **валидно**
- **предсказуемо**
- не **сломается** при взаимодействии **извне**
## Класс и конструктор
[[Конструктор]] - специальный **метод**, который вызывается при **инициализации** объекта и позволяющий правильно настроить его **поля** и **изначальное** **состояние** для дальнейшей работы объекта.
```cs
public class Player
{
    private int _health;

    public Player(int health)
    {
        _health = health;
    }
}
```
Он гарантирует:
- корректное начальное состояние
- задает зависимости
## Статика
Статические члены:
- принадлежат **типу**
- инициализируются **один** раз
- живут до **выгрузки** **домена**

```cs
public class GameSettings
{
    public static int MaxPlayers = 4;
}
```

Важное различие между `Instance` и `Static`:
- `Instance` привязан к **объекту**, а `Static` привязан к **типу**
- `Instance`-ов может быть **много**, а `Static` **один** на весь код
