`List<T>` в [[CSharp|C#]] это коллекция, хранящая элементы, имеющая динамичный размер в отличии от **массивов**, в остальном это тот же массив, но со своими **методами**.
## Основные свойства
1. Допускаются **дубликаты**
   ```cs
   var list = new List<int>();
	list.Add(1);
	list.Add(1); // нормально, добавитcя
   ```
2. **Быстрый** доступ к элементу по **индексу**
   ```cs
   int value = list[5]; // O(1)
   ```
   `List<T>` хранит данные в **массиве**, поэтому он **точно** знает адрес элемента. 
	Это **главное** **преимущество** List, помимо **динамичного** **размера**
3. Медленный поиск по элементу
   ```cs
   list.Contains(10); // O(n)
   ```
  `List<T>` приходится перебирать **все** элементы => скорость зависит от **кол-ва** **элементов** в списке. 
4. Порядок **сохраняется** - элементы хранятся **строго** в порядке **добавления**
     ```cs
     var list = new List<int> { 5, 1, 10 };
	foreach (var value in list)
	{
	    Console.WriteLine(value);
	}
	// 5, 1, 10
     ```
5. **Основные** методы
	```cs
list.Add(item); // добавление в конец - быстро
list.Insert(0, item); // вставка - если в начало/середину - сдвиг всего списка, => долго
list.RemoveAt(index); // сдвигает элементы
list.Remove(item); // ищет + сдвигает
list.Contains(item);
list.IndexOf(item);
list.Count;
list.Clear();
	```
## Когда использовать List
**Подходит**, когда:
- **сохранять** порядок
- работать с **индексами**
- **часто** перебирать элементы
- хранить **дубликаты**
  
**Не подходит**, если:
- **часто** вызывается Contains
- важна **уникальность**
- **большие** **объёмы** данных для **поиска**
