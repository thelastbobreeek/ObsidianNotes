Рекурсия - метод программирования, при котором **функция** вызывает **саму себя**. 

Функция рекурсивная, если она:
- вызывает **сама себя**
- имеет **условие выхода**(базовый случай)

Иначе - **бесконечная** рекурсия -> **StackOverflow**.

Простейший пример:
```cs
int Factorial(int n)
{
    if (n <= 1)
        return 1;        // базовый случай

    return n * Factorial(n - 1);
}
```
Однако, рекурсивную функцию можно **заменить циклом**:
```cs
int FactorialIter(int n)
{
    int result = 1;

    for (int i = 2; i <= n; i++)
        result *= i;

    return result;
}
```
Цикл имеет **менее** читаемый синтаксис, однако:
- память в функции **ограничена**
- скорость **быстрее** чем постоянный вызов функций
- **риска** краша **нет**

Рекурсия **оправдана**, если структура данных **сама рекурсивна**, например:
```cs
void Traverse(Node node)
{
    foreach (var child in node.Children)
        Traverse(child);
}
```
Рекурсия в **большинстве** случаев плоха, но если:
- **небольшая** глубина
- **редко** вызывается
- перечисление объектов **в иерархии**

то её использование **оправдано**
