Свойство в [[CSharp|C#]] - это своего рода интеллектуальное **поле** в классе. 
Элемент, предоставляющий **гибкий** механизм для **чтения**, **записи** или **вычисления** значения **поля** данных, своего рода **"посредник"** между **полями** класса и **внешним** кодом, позволяющий контролировать **доступ** к данным объекта. 

Они выглядят как **поля** **снаружи**, но они реализуются как специальные **методы**. Это позволяет вызывающим классам легко получать доступ к данным и по-прежнему способствовать обеспечению **безопасности** и **гибкости** данных. Можно проверять **правильность**, делать отложенное вычисление и т.д.
## Синтаксис свойства
```cs
class Person
{
    private string _name; // приватное поле

    // Свойство для доступа к _name
    public string Name
    {
        get { return _name; }    // чтение
        set { _name = value; }   // запись
    }
}
```
- `get` вызывается при **чтении** свойства(person.Name)
- `set` вызывается при **присвоении** значения(person.Name = "Tim"), `value` - **значение** которое **присваивают**
## Автоматические свойства
```cs
public class Person
{
    public string Name { get; set; } // автоматически создается скрытое поле
}
```
При объявлении автоматического свойства [[Компилятор|компилятор]] создает **скрытое** **поле** резервного копирования для свойства. 
Выглядит это примерно так
```cs
private string _name;

public string Name
{
    get { return _name; }
    set { _name = value; }
}
```
**Снаружи** выглядит как **поле**, но **внутри** - **методы**. Можно добавить **логику**, не меняя внешний вид других классов.
## Управление доступом
Предположим, что класс `Person` должен изменять значение свойства `FirstName` **только** **в** **своих** методах. В таком случае стоит использовать **ограничивающий** **модификатор** для аксессора `get` или `set`. 
```cs
public class Person
{
    public string? FirstName { get; private set; }
}
```
Модификатор должен быть **более** **строгим** к **аксессору**, чем **доступ** к **свойству**. 
Предыдущий код является **законным**, так как свойство `FirstName` является **публичным**, но свойство set является `private`. Нельзя объявить `private` свойство с `public` аксессором. 
Свойство также можно объявить как `protected`, `internal`, `protected internal` или даже `private`(см. [[Модификаторы доступа]]).

Существуют два **специальных** **модификатора** для `set` аксессора:
- `init` - его можно вызвать **только** из **инициализатора** **объекта** или **конструктора** типа. Это **более** **строго**, чем `private`.
- Автоматически реализованное свойство может объявлять `get` аксессор **без** `set` аксессора. В этом случае компилятор позволяет вызывать аксессор **только** из **конструктора** типа. Это еще **более** **ограничительно**, чем `init`
## Обязательные свойства
В обычном объявлении свойства можно создать экземпляр объекта с этим свойством **без** **установления** **значения** свойства. Однако, можно **потребовать**, чтобы вызывающие пользователи **обязательно** **задавали** свойство:
```cs
public class Person
{
    public Person() { }

    [SetsRequiredMembers]
    public Person(string firstName) => FirstName = firstName;

    public required string FirstName { get; init; }
}
```
`FirstName` объявление свойства включает `required` модификатор. Это означает, что любой код, создающий новое Person свойство, **должен** **задать** это свойство с помощью **инициализатора** **объектов**.

Важно **не** **путать** `required` с **ненулевой**. **Допустимо** задать свойству `required` значение `null`. Если тип **не** **допускает** значения `null`, как например `string`, то компилятор выдаст **предупреждение**.
