Конкатенация строк в [[CSharp|C#]] - **процесс добавления одной строки в конец другой**. 

Сцепление происходит с помощью операторов + и +=.  Для **строковых литералов** и **строковых констант** объединение происходит на **моменте компиляции**. Для строковых **переменных** же объединение происходит **во** **время выполнения**.
## Операторы + и +=
Для **конкатенации строк** используются операторы **+ и +=.** 

Даже если в одной инструкции используется **несколько** операторов +, содержимое строки **копируется только один раз**.

Например, при инструкции(выражении) вида: 
```cs
string s = a + b + c + d;
```
НЕ происходит:
```cs
t1 = a + b //копирование(создание нового объекта + ссылка на него)
t2 = t1 + c //еще копирование
t3 = t2 + d //и еще копирование
```
Это был бы ад по производительности. 

На самом деле же компилятор **видит** цепочку из + в одном выражении и **методом string.Concat():**
- вычисляет итоговую длину
- **один** раз выделяет память под результат
- **один** раз копирует все куски подряд

Поэтому и говорится "копируется только **один раз**"
### + при конкатенации строк
```cs
string a = "Hello";
string b = a + " world";
```
Что происходит: 
- создается **новая строка**
- **копируется** содержимое a
- добавляется " world"
### += при конкатенации строк
``` cs 
string a = "Hello";
a += " world";
```

+= делает то же самое, что и + с присваиванием:
``` cs
a = a + "world"
```
То есть: 
- старая строка "Hello" остается в памяти, пока на неё кто-то ссылается, затем её удалит [[Сборщик мусора|сборщик мусора]]
- создается **новый объект** типа string
- ссылка переменной **a** начинает указывать **на него**
### Между + и += разницы нет
+= в C#:
``` cs
a += b
```
это тоже самое что и:
``` cs
a = a + b
```
Он **не добавляет** к концу существующей строки некие символы. Он **создает** **еще один** объект string, а переменная a начинает **на него** **ссылаться**. То же самое что и оператор +.

Ни по **результату**, ни по **производительности** или [[Intermediate Language|IL коду]] разницы нет.

Абстрактная разница только в **читаемости кода** и в **предпочтениях программиста**.
## Любая строка неизменяема. Любое изменение содержимого строки или "добавление" в неё символов создает новый объект строки, на который начинает ссылаться "изменённая" переменная типа string.
