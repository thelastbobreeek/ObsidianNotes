Конструктор в [[CSharp|C#]]- это **метод**, вызываемый при **создании** экземпляра класс или структуры. Конструктор может иметь [[Перегрузка|перегрузки]] - **несколько** вариаций в зависимости от **аргументов**. Конструкторы позволяют обеспечить **корректность** экземпляров типа, когда они создаются.
## Синтаксис конструктора
Конструктор - метод с **тем** **же** **именем**, что и его **класс**. Его сигнатура может включать необязательный [[Модификаторы доступа|модификатор доступа]], имя метода и список параметров. Он **не** **включает** тип **возвращаемого** **значения**. Вот пример конструктора для класса `Person`
```cs
public class Person
{
   private string last;
   private string first;

   public Person(string lastName, string firstName)
   {
      last = lastName;
      first = firstName;
   }

   // Remaining implementation of Person class.
}
```
### Наследование в конструкторе
Для вызова конструктора **родителя** стоит использовать **ключевое** **слово** `base`.

Если есть **наследование**:
```cs
class Character
{
    protected int _health;

    public Character(int health)
    {
        _health = health;
    }
}
```
И есть **наследник**:
```cs
class Player : Character
{
    public Player(int health) : base(health)
    {
    }
}
```
Происходит следующее:
- `Characted` **не** **имеет** пустого конструктора
- Поэтому `Player` **обязан** вызвать конструктор **родителя**
- Делается это через `: base(...)` 
- Если не вызвать `base(health)` - код **не** **скомпилируется**!

`Player player = new Player(100);`
Происходит же всё в таком порядке:
1. Сначала вызывается `Character(int health)`
2. Потом - `Player(int health)`

## Как работает `base` и когда его  писать
#### Если у родителя нет ни одного конструктора:
```cs
class Parent
{
}
```
То компилятор сам его создает:
```cs
public Parent()
{
}
```
Пустой конструктор всегда существует, независимо от того, написан он в коде или нет.

Если есть наследник:
```cs
class Child : Parent
{
    public Child()
    {
    }
}
```
Компилятор превращает его конструктор в:
```cs
public Child() : base()
{
}
```
И все работает
#### Если у родителя есть пустой конструктор:
```cs
class Parent
{
    public Parent()
    {
        Console.WriteLine("Parent");
    }
}
```
И есть наследник:
```cs
class Child : Parent
{
    public Child()
    {
        Console.WriteLine("Child");
    }
}
```
Создаётся: `new Child();` 

Порядок выполнения:
1. Сначала - конструктор `Parent()`
2. Потом - `Chil()`
Компилятор снова добавляет `: base()` автоматически.
В таком случае явно писать `: base()` нет смысла, так как компилятор и так его добавляет по дефолту.
#### Если у родителя есть конструктор с параметрами
```cs
class Parent
{
    public Parent(int value)
    {
    }
}
```
И есть наследник:
```cs
class Child : Parent
{
    public Child()
    {
    }
}
```
Компилятор пытается сделать
```cs
public Child() : base()
{
}
```
Но `base()` не существует в родительском классе - есть только `base(int value)`. Это приведет к ошибке компиляции.

Правильный вариант:
```cs
class Child : Parent
{
    public Child() : base(10)
    {
    }
}
```
Тут явно говорится, какой конструктор родителя надо использовать. По аналогии, если будет еще и конструктор с аргументом типа `string`, надо будет указать, какой использовать - `string` или `int`.
> [!note]  Если в родительском классе написан хотя бы один конструктор, то компилятор больше не создаёт пустого автоматически.
