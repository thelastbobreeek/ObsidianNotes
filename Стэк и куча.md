Стэк и куча и то и то **область памяти**, но с разными механизмами **распределения** и **управления** памяти. 
## Стэк
Стэк - область памяти, работающая по принципу **LIFO(last in first out)**.

Используется в первую очередь для:
- локальных **переменных** функций
- **аргументов** функций
  
Создается **автоматически** при входе в функцию и **уничтожается** при **выходе** из неё.
```cs
void Foo()
{
    int x = 10; // в стеке
}
```
- x существует **пока выполняется Foo**
- как только функция завершилась - память освобождена
- быстро и безопасно

Стэк имеет **ограниченный размер**(обычно несколько МБ), при переполнении размера - StackOverflow. Например,
```cs
void Boom()
{
	int[] array = new int[10_000_000] // гарантированный StackOverflow
}
```

## Куча
Куча - область памяти для **динамического выделения**, где объекты живут, пока на них кто-то **ссылается**(или пока их явно не освободят)

Используются для:
- объектов
- массивов **переменной** длины
- данных с **непредсказуемым** временем жизни

Удаляется вручную в C/C++ или через [[Сборщик мусора|GC]] (C#,Java,etc)
```cs
class Player { public int Hp; }

void Foo()
{
    Player p = new Player(); // объект в куче
}
```
- p в стэке(это **локальная переменная**, по сути ссылка(0xA13F...))
- Player - **объект**, лежит в где-то в куче
- **объект** Player живёт пока на него есть **хотя бы одна** **ссылка**
  
После выполнения Foo в данном случае **p удалится**, а объект, на который она указывала, станет **garbage**, который [[Сборщик мусора|GC]] когда-нибудь удалит. 

Может использовать почти **всю доступную RAM**, благодаря **динамическому** размеру.
