Полиморфизм - **возможность** объектов с **одинаковой спецификацией** иметь **различную** реализацию. Наследник **может** выполнять команду базового класса **по-своему**.  

Например, объект типа **Animal** может хранить в себе объекты наследника Dog, Horse, Cow.
``` cs
Animal a1 = new Dog();
Animal a2 = new Cat();

a1.Speak(); //гав
a2.Speak(); //мяу
```

Несмотря на то, что **обе** переменные объявлены как **Animal**, вызывается метод именно того класса, что **реально** лежит внутри переменной.

Это и есть **полиморфизм** - разные объекты ведут себя **по-разному**, но вызываются **единым** способом. 

**Полиморфизм** используется для принятия базового типа в метод:
``` cs
public void MakeAnimalSpeak(Animal animal)
{
	animal.Speak(); //у каждого свой звук
}
```
Также для написания общего кода для разных типов:
``` cs
Animal[] animals = { new Dog(), new Cat(), new Dog() };

foreach (var animal in animals)
{
    animal.Speak();  // каждый выведет свой звук
}
```
